# TRON / USDT GPU 靓号钱包工具

> 专为波场 TRX / USDT 场景打造的 GPU 靓号生成程序  
> 功能覆盖：高仿地址、连号地址、自定义前后缀、顺子号、6~10A 等多种靓号形态  
> 支持离线运行，适合本地电脑、GPU 服务器长期挂机使用。

---

## 1. 工具简介

这是一个利用显卡算力，对 TRON 私钥空间进行高速遍历、从中筛选“好看地址”的程序。  
你可以用它来：

- 生成用于收款展示的 TRX / USDT 靓号钱包  
- 批量搭建“靓号地址池”（配合广告、空投、地址展示等场景）  
- 生成特定模式的高仿地址（前 N 位 / 后 N 位与目标地址一致）  
- 搜索带有连续数字、连续字母、重复字符等“易记地址”  

程序为绿色版，无需额外复杂依赖，准备好显卡驱动即可启动。

> ⚠️ 温馨提示：  
> 本工具偏向密码学与地址算法研究，请合理使用。  
> 生成的钱包若用于真实资产，请务必做好冷备 & 多重备份。

## 3. 功能亮点

### 3.1 GPU 并行爆破

- 支持多线程 + GPU 内核（CUDA / OpenCL）  
- 实测 RTX 2060 速度约：**135 MH/s 左右**（即每秒尝试 ~1.35 亿个地址）  
- 更高算力显卡 / 多卡并行时，搜索速度可进一步提升

### 3.2 丰富的靓号规则

你可以自由组合以下匹配方式：

- **前缀规则**：  
  - 指定地址开头，如 `TTTRX8***`、`TUSDT*888` 等样式  
- **后缀规则**：  
  - 固定尾号，例如 `******888888`、`******123456`  
- **连号规则**：  
  - 支持 6A / 7A / 8A / 9A / 10A 等各种连续重复字符  
- **高仿模式**：  
  - 以真实地址为基准，生成前 N 位 + 后 N 位几乎一致的“仿生地址”  
- **顺子模式**：  
  - 识别连续升序/降序组合：`123456`、`234567`、`ABCDE`、`CDEF01` 等  
- **自定义模式**：  
  - 通过配置文件指定复杂的匹配模板（前/后各 10 位的自由组合）

### 3.3 结果持久化与分类保存

- 实时将命中的地址（及其私钥等信息）写入本地 txt 文件  
- 自动按“类型”拆分存储，例如：  
  - 连号类、顺子类、高仿类、混合类等  
- 支持中途关闭程序，已有结果不会丢失

### 3.4 输出字段完整

- TRON 地址（Base58Check，T 开头）  
- 对应私钥（HEX 格式，部分版本支持 WIF）  
- 可选导出公钥等辅助字段  
- 保存生成时使用的参数，以便后续核对

### 3.5 多平台可用

- **Windows**：10 / 11 / Server 系列均可  
- **Linux**：多种主流发行版，只需自行安装显卡驱动  
- **macOS**：以 CPU 模式为主，不建议用于长时间大规模爆破  
- 支持物理机 / 虚拟机 / GPU 云服务器等多种环境

## 4. 工作机理简要说明

TRON 地址的生成过程，可以理解为下面几个步骤：

1. 随机选取一个 secp256k1 私钥 `sk`；  
2. 基于椭圆曲线算法计算出公钥 `pk`；  
3. 对公钥进行 `keccak256` 哈希，获取后 20 字节作为核心地址内容；  
4. 在前面增加波场主网前缀 `0x41`，组成 21 字节二进制地址；  
5. 将该 21 字节数据做 Base58Check 编码（含双 SHA256 校验）；  
6. 得到一个以 `T` 开头的 TRON 地址字符串；  
7. 用你设置好的匹配规则对这个地址进行判断：  
   - 如果满足规则 → 写入对应的结果文件  
   - 不满足 → 丢弃，继续下一组随机私钥

不断重复上述操作，即形成“爆破靓号地址”的过程。

## 5. 关键文件说明

### 5.1 主程序

- `gd_gpu.exe`  
  - 在 Windows 下的执行文件（Linux / 其他平台名称会有差异）  
  - 建议与配置文件放在同一目录，方便共享参数

### 5.2 `config.txt`

- 顶部数行数字用于控制内部“随机基数”与“采样节奏”  
- 通过调整这些数值，可以改变高难度靓号（如 9A、10A）的出现频率  
- 不要求用户理解其数学含义，只需：  
  - 想换一批“随机轨迹”时，改一改数字重新启动程序即可

### 5.3 `address.txt`

- 用来存放需要“高仿”的目标地址  
- 一行一个真实地址  
- 结合程序参数中指定的 `qian`（前缀长度）和 `hou`（后缀长度）一起使用：  
  - 如：前 3 + 后 4、前 4 + 后 6 等

### 5.4 `diy.txt`

- 自定义模式配置文件  
- 每一行对应一个“前 10 位 + 后 10 位”的模式组合，总体 20 位长度  
- 示例思路：  
  - 只固定后缀：前 10 写占位（比如全 T），后 10 写心仪尾号  
  - 同时固定前后：前 10 写自有前缀，后 10 写想要的尾数

### 5.5 启动脚本（Windows）

- `点击运行.bat`  
- `高仿地址.bat` 等  
- 本质只是封装了一条命令行执行指令，例如：

```bat
gd_gpu.exe --lianhao 7 --down nice_7A.txt
